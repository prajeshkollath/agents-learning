# AI Agents MOC

**Top-level hub** for everything I learn about AI agents — the 12 concept domains
interviewers test for agentic AI engineer / architect roles. Each domain is its
own note that links to its sub-topics, so the graph is a clean hierarchy:

`AI Agents MOC → 12 domains → individual concept notes`

Open **graph view** to see it. Domains in suggested learning order:

## Domains
1. [[01 Agent Fundamentals]] 🧠 — agentic vs LLM chain, the agent loop, autonomy boundaries
2. [[02 Agent Architecture and Orchestration]] 🏗️ — orchestrator vs LLM, safe loops, termination
3. [[04 Tool Use and Function Calling]] 🔧 — tool selection, schemas, failure handling, sandboxing
4. [[03 Memory Systems]] 🧩 — working / episodic / semantic / procedural memory
5. [[05 RAG and Knowledge Grounding]] 📚 — chunking, hybrid search, re-ranking, grounding
6. [[06 Multi-Agent Systems]] 🤖 — supervisor/subagent, peer-to-peer, coordination
7. [[10 Prompt Engineering]] ⚡ — few-shot, CoT, ReAct, structured output, versioning
8. [[08 Evaluation]] 📊 — trajectory vs outcome evals, LLM-as-judge, metrics
9. [[09 Observability and Debugging]] 🔭 — tracing, logs/metrics, root-cause analysis
10. [[07 Safety Guardrails and Reliability]] 🛡️ — fail-safe defaults, policy engine, PII, injection
11. [[12 Production and Cost Engineering]] 💼 — token cost, caching, latency
12. [[11 LLM Internals]] 🏛️ — attention, context limits, Mamba/SSMs, model selection

> Numbering follows the source domains (1–12); the list order above is the
> suggested study sequence (tools before memory, internals last).
